# 教学对话助手 — 与模型的交互方式

## 一、LLM 客户端（`services/llm/client.go`）

底层统一调用 **DeepSeek API**（OpenAI 兼容接口），通过 `config.AppConfig.DeepSeek` 配置：

```go
type Client struct {
    APIKey  string
    BaseURL string  // 如 https://api.deepseek.com/v1
    Model   string  // 如 deepseek-chat
    Client  *http.Client
}
```

## 二、非流式调用：`Chat(messages, tools)`

发送 `POST {BaseURL}/chat/completions`，`stream: false`：

```go
httpReq.Header.Set("Authorization", "Bearer "+c.APIKey)
// 返回完整 ChatResponse
```

用于备课工作流中大纲/教案/PPT 结构生成（需要完整 JSON，不适合流式解析）。

## 三、流式调用：`StreamChat(messages, tools, onChunk)`

发送同一接口，`stream: true`，`Accept: text/event-stream`，用 `bufio.Scanner` 逐行读 SSE：

```go
for scanner.Scan() {
    line := scanner.Text()
    if !strings.HasPrefix(line, "data: ") { continue }
    data := strings.TrimPrefix(line, "data: ")
    if data == "[DONE]" { break }
    json.Unmarshal([]byte(data), &chunk)
    onChunk(&chunk)      // 回调给上层（写 SSE 到客户端）
    contentBuilder.WriteString(chunk.Choices[0].Delta.Content)
}
```

`onChunk` 回调由业务层提供，对话助手中的实现（`controllers/dialog.go:107`）：

```go
func(chunk *llm.StreamChunk) error {
    if delta.Content != "" {
        sendSSE(c, SSEEvent{Type: "text", Content: delta.Content})
    }
    if len(delta.ToolCalls) > 0 {
        for _, tc := range delta.ToolCalls {
            sendSSE(c, SSEEvent{Type: "tool_call", Tool: tc.Function.Name})
            result, _ := d.toolExecutor.ExecuteTool(tc.Function.Name, tc.Function.Arguments)
            sendSSE(c, SSEEvent{Type: "tool_result", Tool: tc.Function.Name, Result: result})
        }
    }
    return nil
}
```

## 四、工具调用（Function Call）

### 工具注册（`services/llm/client.go:251`）

`BuildTools(mode)` 根据 mode 返回工具列表，工具以 OpenAI Function Calling 格式定义：

```go
type Tool struct {
    Type     string       `json:"type"`     // "function"
    Function ToolFunction `json:"function"`
}

type ToolFunction struct {
    Name        string                 `json:"name"`
    Description string                 `json:"description"`
    Parameters  map[string]interface{} `json:"parameters"` // JSON Schema
}
```

**注册的工具：**

| 工具名 | 描述 | Mode |
|--------|------|------|
| `search_knowledge` | 知识库全文/模糊检索，支持 7 种 category | 所有 |
| `query_graph` | 知识图谱实体关系查询，支持 relation_type 过滤 | 所有 |
| `get_timeline` | 时间线事件查询（按年份/国家） | 所有 |
| `web_search` | Serper API 联网搜索 | online/auto |

### 工具执行（`services/mcp/tools.go`）

`ToolExecutor.ExecuteTool(name, args json.RawMessage)` switch 分发：

```go
case "search_knowledge": → rag.Retriever.Search(query, category, limit)
case "query_graph":      → graph.QueryService.QueryByName / SearchEntities
case "get_timeline":     → 返回 TODO 占位 JSON
case "web_search":       → callSerperAPI(query, apiKey)
```

所有工具均返回 JSON 字符串，由 LLM 读取后继续生成最终回答。

## 六、流式输出 → SSE 推送链路

```
DeepSeek SSE stream
    ↓
bufio.Scanner 逐行读
    ↓
json.Unmarshal → StreamChunk
    ↓
onChunk 回调
    ↓
fmt.Fprintf(c.Writer, "data: %s\n\n", json)
c.Writer.(http.Flusher).Flush()
    ↓
浏览器 EventSource 接收
```

Header 设置（`controllers/dialog.go:61`）：
```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
Access-Control-Allow-Origin: *
```

## 七、DeepSeek vs 大模型网关

| 维度 | 当前实现 | 可扩展方向 |
|------|---------|-----------|
| 模型 | DeepSeek（单一） | 多模型路由，本地优先云端兜底 |
| 重试 | 无 | 指数退避重试（超时/连接失败）|
| 熔断 | 无 | 按 provider 失败率熔断 |
| 可观测 | 无 TraceID | 每次请求生成 TraceID 串联 RAG/MCP/LLM 链路 |

> 当前以最小依赖直接调用 DeepSeek API；大规模场景可在 `llm.Client` 上层封装路由层。
