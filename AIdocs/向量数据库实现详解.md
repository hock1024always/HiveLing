## 概述

本项目在原有 MySQL 关系型数据库基础上，引入了 Milvus 向量数据库来增强语义搜索能力。通过将文本转换为高维向量，系统能够理解用户查询的真实意图，实现更智能的知识检索。

## 向量数据库基础原理

### 什么是向量？

向量是数学中的基本概念，在机器学习中用来表示数据的数值化特征：

```
文本: "齐桓公是春秋时期的霸主"
↓ Embedding 模型
向量: [0.12, -0.34, 0.56, 0.78, ..., -0.23]  // 1536维
```

### 向量的语义特性

向量具有重要的语义特性：
- **语义相近的文本 → 向量距离近**
- **语义不同的文本 → 向量距离远**

```
"齐桓公"    → [0.12, -0.34, 0.56, ...]
"姜小白"    → [0.13, -0.32, 0.58, ...]  // 同一人，向量接近
"秦始皇"    → [0.89,  0.45, -0.12, ...] // 不同时期，向量较远
```

### 向量距离计算

常用的向量距离度量方法：

1. **余弦相似度** (Cosine Similarity)
   ```
   cos(θ) = (A·B) / (||A|| × ||B||)
   
   结果范围: [-1, 1]
   1 表示完全相同，-1 表示完全相反
   ```

2. **欧几里得距离** (L2 Distance)
   ```
   distance = √[(a₁-b₁)² + (a₂-b₂)² + ... + (aₙ-bₙ)²]
   
   距离越小，向量越相似
   ```

3. **内积** (Dot Product)
   ```
   A·B = a₁×b₁ + a₂×b₂ + ... + aₙ×bₙ
   ```

### 向量数据库 vs 关系型数据库

| 特性 | 关系型数据库 (MySQL) | 向量数据库 (Milvus) |
|------|---------------------|--------------------|
| 存储内容 | 结构化数据 | 高维向量 + 标量 |
| 查询方式 | 精确匹配、范围查询 | 相似度搜索 |
| 索引类型 | B+ Tree、Hash | IVF、HNSW、ANNOY |
| 查询结果 | 完全匹配的记录 | 最相似的 K 个结果 |
| 适用场景 | 事务处理、精确查询 | 语义搜索、推荐系统 |
| 查询语义 | "找完全一样的" | "找意思相近的" |

### Go 代码实现

```go
func (m *MilvusClient) InsertVectors(vectors []KnowledgeVector) error {
    // 准备列式数据
    ids := make([]int64, len(vectors))
    titles := make([]string, len(vectors))
    contents := make([]string, len(vectors))
    categories := make([]string, len(vectors))
    vecs := make([][]float32, len(vectors))
    
    for i, v := range vectors {
        ids[i] = v.ID
        titles[i] = truncateString(v.Title, 500)
        contents[i] = truncateString(v.Content, 8000)
        categories[i] = v.Category
        vecs[i] = v.Vector
    }
    
    // 构造列
    columns := []entity.Column{
        entity.NewColumnInt64("id", ids),
        entity.NewColumnVarChar("title", titles),
        entity.NewColumnVarChar("content", contents),
        entity.NewColumnVarChar("category", categories),
        entity.NewColumnFloatVector("vector", m.dimension, vecs),
    }
    
    // 批量插入
    _, err := m.client.Insert(ctx, m.collection, "", columns...)
    if err != nil {
        return fmt.Errorf("failed to insert vectors: %v", err)
    }
    
    // 刷新确保可见
    m.client.Flush(ctx, m.collection, false)
    return nil
}
```

## 索引优化原理

### 为什么需要索引？

```
┌─────────────────────────────────────────────────────────────────┐
│  暴力搜索（慢）                                                  │
│  ────────────────                                              │
│  遍历所有向量，计算距离                                          │
│  100万条数据 → 100万次距离计算                                   │
│  时间复杂度: O(N)                                               │
├─────────────────────────────────────────────────────────────────┤
│  ANN 索引（快）                                                  │
│  ────────────────                                              │
│  先聚类/分桶，只搜索可能相似的子集                                │
│  100万条数据 → 只需搜索几千条                                    │
│  时间复杂度: O(log N)                                           │
└─────────────────────────────────────────────────────────────────┘
```

### AUTOINDEX 策略

```go
// Milvus 自动选择最优索引算法
idx, err := entity.NewIndexAUTOINDEX(entity.L2)

// 可选的索引类型：
// ┌─────────────┬───────────────────────────────────────┐
// │ IVF         │ 聚类索引，将向量分桶                   │
// │ HNSW        │ 层次导航小世界图，近似最近邻搜索        │
// │ ANNOY       │ 随机投影树，适合高维向量               │
// └─────────────┴───────────────────────────────────────┘
```

### 索引构建过程

```
原始向量数据
    │
    ▼
聚类算法 (如 K-means)
    │
    ▼
分成多个簇 (buckets)
    │
    ▼
建立倒排索引
    │
    ▼
搜索时只需查相关簇
```

## 混合检索策略

### 设计理念

结合向量检索和关键词检索的优势：

```go
// 混合检索 = 向量检索 × 权重 + 关键词检索 × 权重
final_score = vector_score × 0.6 + keyword_score × 0.4
```

### 实现流程

```

         │
         ▼
```

### Go 代码实现

```go
func (r *Retriever) mergeResults(vectorResults, keywordResults []SearchResult, limit int) []SearchResult {
    vectorWeight := 0.6
    keywordWeight := 0.4
    
    mergedMap := make(map[uint]*SearchResult)
    
    // 处理向量检索结果
    for _, result := range vectorResults {
        id := result.Chunk.ID
        if existing, ok := mergedMap[id]; ok {
            existing.Score = existing.Score*keywordWeight + result.Score*vectorWeight
            existing.Source = "hybrid"
        } else {
            result.Score = result.Score * vectorWeight
            result.Source = "vector"
            mergedMap[id] = &result
        }
    }
    
    // 处理关键词检索结果
    for _, result := range keywordResults {
        id := result.Chunk.ID
        if existing, ok := mergedMap[id]; ok {
            if existing.Source == "vector" {
                existing.Score = existing.Score + result.Score*keywordWeight
                existing.Source = "hybrid"
            }
        } else {
            result.Score = result.Score * keywordWeight
            result.Source = "keyword"
            mergedMap[id] = &result
        }
    }
    
    // 排序并返回
    results := make([]SearchResult, 0, len(mergedMap))
    for _, result := range mergedMap {
        results = append(results, *result)
    }
    
    sort.Slice(results, func(i, j int) bool {
        return results[i].Score > results[j].Score
    })
    
    if len(results) > limit {
        results = results[:limit]
    }
    
    return results
}
```

### 缓存策略

| 缓存类型 | 有效期 | 说明 |
|---------|-------|------|
| Embedding 缓存 | 24 小时 | 文本向量不会变化，长期缓存 |

### 缓存实现代码

```go
// 检索结果缓存示例
func (r *Retriever) Search(query string, category string, limit int) ([]SearchResult, error) {
    // 1. 尝试从缓存获取
    if r.cacheEnabled {
        ctx := context.Background()
        cacheKey := cache.SearchCacheKey(query, category, limit, r.useVectorSearch)
        var cachedResults []SearchResult
        if err := r.cacheSvc.Get(ctx, cacheKey, &cachedResults); err == nil {
            // 缓存命中
            for i := range cachedResults {
                cachedResults[i].Source = "cache"
            }
            return cachedResults, nil
        }
    }

    // 2. 缓存未命中，执行检索
    results, err := r.executeSearch(query, category, limit)
    if err != nil {
        return nil, err
    }

    // 3. 异步缓存结果
    if r.cacheEnabled && len(results) > 0 {
        go func() {
            bgCtx := context.Background()
            cacheKey := cache.SearchCacheKey(query, category, limit, r.useVectorSearch)
            r.cacheSvc.Set(bgCtx, cacheKey, results, 10*time.Minute)
        }()
    }

    return results, nil
}
```
