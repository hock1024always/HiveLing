# 指代消解（Coreference Resolution）技术方案

> 适用项目：HiveLing AI教育平台 - 春秋战国历史问答系统
> 场景：多轮对话中，用户使用代词（"他"、"该国"）或省略主语时，系统自动还原语义

---

## 一、问题定义

### 什么是指代消解？

指代消解是 NLP 中一个经典问题：**识别文本中代词/名词短语所指向的具体实体**。

在多轮历史问答场景中，典型的指代情况包括：

```
第1轮：用户："介绍一下齐桓公"
第2轮：用户："他是怎么称霸的？"  ← "他" 指代 "齐桓公"

第1轮：用户："秦国是如何变法的？"
第2轮：用户："该国后来统一六国用了多久？"  ← "该国" 指代 "秦国"

第1轮：用户："商鞅变法的内容有哪些？"
第2轮：用户："后来呢？"  ← 省略主语，需要补全"商鞅"或"商鞅变法"
```

### 为什么难处理？

1. **歧义性**：对话中可能同时出现多个实体，代词指向不明确
2. **跨轮距离**：代词和实体可能相距多轮对话
3. **零代词**（省略）：中文常见主语省略，没有显式代词
4. **隐式指代**：如"这场战争"需要理解上下文才能确定是哪场战争
5. **多人关系**："他们之间有什么矛盾？"涉及多个实体

## 二、整体方案设计

### 架构：规则优先 + LLM 兜底

```
用户输入 Query
      ↓
[规则消解器]
  - 代词词典匹配
  - 实体栈查找
  - 省略补全
      ↓
[置信度评估]
  confidence >= 0.85 ──→ 直接返回结果
  confidence < 0.85  ──→ [LLM消解器]
      ↓                        ↓
  快速路径              精确路径（带对话历史）
                               ↓
                         合并最终结果
                               ↓
                     消解后的完整查询 Query'
                               ↓
                       后续 RAG 检索
```

**设计原则**：
- 简单情况走规则，速度快（<1ms）
- 复杂情况走 LLM，精度高（但有延迟）
- 两层降级保障，LLM 失败时回退规则结果

## 三、核心组件详解

### 3.1 实体栈（EntityStack）

实体栈是指代消解的核心数据结构，用于跟踪对话中提到的实体：

```go
type EntityStack struct {
    persons  []EntityMention  // 人物实体栈（如：齐桓公、孔子）
    states   []EntityMention  // 国家实体栈（如：秦国、楚国）
    events   []EntityMention  // 事件实体栈（如：商鞅变法、长平之战）
    times    []EntityMention  // 时间实体栈（如：春秋时期、公元前770年）
    maxSize  int              // 最大保留数量（默认10）
}

type EntityMention struct {
    Text      string  // 实体文本
    Type      string  // 实体类型
    Turn      int     // 出现的对话轮次
    Position  int     // 在句子中的位置
    Frequency int     // 提及次数
}
```

**更新策略**：
- 每轮对话结束后，用 jieba 提取实体并压入栈
- 如果实体已存在，更新频率计数和轮次，并移到栈顶（最近原则）
- 栈满时丢弃最旧的实体（LRU 策略）

**查找策略**：
- 只查找比当前轮次更早的实体（避免自引用）
- 栈顶元素优先（最近提到的实体优先级最高）

### 3.2 代词词典

针对中文历史问答场景，建立专用代词词典：

```
人称代词 → 优先匹配 person 类型实体
  "他" / "她" / "此人" / "该人" / "这位" / "那位" / "其"

国家代词 → 优先匹配 state 类型实体  
  "该国" / "此国" / "这个国家" / "那个国家"

事件代词 → 优先匹配 event 类型实体
  "这场战争" / "那场战争" / "此事" / "该事件"

时间代词 → 优先匹配 time 类型实体
  "这个时期" / "那个时期" / "当时" / "那时"

通用指示代词 → 按实体类型优先级匹配
  "这个" / "那个" / "此" / "该"
```

### 3.3 省略补全（Zero Pronoun）

中文中主语省略极为普遍，规则如下：

| 省略模式 | 示例 | 处理方式 |
|---------|------|---------|
| 动词起头 | "是怎么称霸的？" | 在句首补全最近人物实体 |
| 转折接续 | "后来呢？" / "然后呢？" | 补全上一轮的主语实体 |
| 比较引导 | "和晋文公比呢？" | 补全上一轮的比较对象 |
| 时间疑问 | "什么时候灭亡的？" | 补全最近的国家/事件实体 |

### 3.4 置信度评估

```
基础置信度 = 1.0

每次代词替换  × 0.90
省略主语补全  × 0.85

threshold = 0.85：高置信度，直接使用规则结果
threshold = 0.70：中等置信度，尝试 LLM 验证
threshold < 0.70：低置信度，必须使用 LLM 处理

触发 LLM 的其他条件：
- 存在多人关系代词："他们之间"
- 存在对比代词："前者" / "后者" / "两者"
- 替换次数 > 2
```

### 3.5 LLM 消解（Fallback）

当规则消解置信度不足时，调用 LLM 处理：

**Prompt 设计**：
```
你是一个专业的中文指代消解专家。请分析用户的当前问题，
根据对话历史，将其中的代词（如"他"、"她"、"这个"等）
替换为具体的实体名称。

对话历史：
{recent_5_turns}

当前问题：{query}

输出格式：
{
  "resolved_query": "替换后的完整问题",
  "entities": {"代词": "实体"},
  "reasoning": "推理过程"
}
```

**关键设计点**：
- 只传入最近 5 轮对话（控制 token 消耗）
- 要求输出结构化 JSON（易于解析）
- 要求输出推理过程（便于调试）
- LLM 失败时降级到规则结果，不影响主流程

## 四、集成到查询优化器

指代消解作为 QueryOptimizer 的第一个处理步骤：

```go
func (o *QueryOptimizer) OptimizeQueryWithContext(
    query string, 
    dialogHistory []string, 
    turn int,
) (*QueryAnalysis, error) {

    // Step 1: 指代消解（最先执行）
    corefResult := o.corefResolver.Resolve(query, dialogHistory, turn)
    workQuery := corefResult.ResolvedQuery  // 消解后的查询

    // Step 2: 实体提取（基于消解后的查询）
    analysis.Entities = o.extractEntities(workQuery)

    // Step 3: 意图识别
    analysis.Intent = o.identifyIntent(workQuery)

    // Step 4: 时间范围解析
    analysis.TimeRange = o.parseTimeRange(workQuery)

    // Step 5: 查询扩展
    analysis.ExpandedQueries = o.expandQuery(workQuery, analysis.Entities)

    // Step 6: 更新实体栈（供下一轮使用）
    o.corefResolver.AddContext(extractedEntities, turn)

    return analysis, nil
}
```

**为什么指代消解要最先执行？**

因为后续的实体提取、意图识别、查询扩展都依赖于完整的查询语义。如果 "他" 没有被还原为 "齐桓公"，那么实体提取就无法识别出正确的人物实体，检索结果也会偏差。

## 五、效果示例

### 示例 1：人称代词消解

```
对话历史：
  Turn 1 - 用户: "介绍一下齐桓公"
  Turn 1 - 系统: "齐桓公，姜姓，名小白，是春秋时期..."

Turn 2 - 用户输入: "他是怎么称霸的？"

指代消解流程：
  1. 检测到代词 "他" → 类型 person
  2. 查找实体栈 persons → 栈顶：EntityMention{Text:"齐桓公", Turn:1}
  3. 当前轮次 turn=2 > 实体轮次 1，可以使用
  4. 替换 "他" → "齐桓公"
  5. 置信度：1.0 × 0.9 = 0.9（高于阈值，使用规则结果）

消解结果: "齐桓公是怎么称霸的？"
置信度: 0.90
```

### 示例 2：省略主语补全

```
对话历史：
  Turn 1 - 用户: "商鞅变法的内容有哪些？"
  Turn 1 - 系统: "商鞅变法主要包括..."
  Turn 2 - 用户: "对秦国的影响是什么？"
  Turn 2 - 系统: "商鞅变法使秦国迅速强大..."

Turn 3 - 用户输入: "后来呢？"

指代消解流程：
  1. 检测到省略模式 "^后来(呢|怎么样|如何)"
  2. 查找实体栈 persons → 栈顶：EntityMention{Text:"商鞅", Turn:1}
  3. 查找实体栈 events → 栈顶：EntityMention{Text:"商鞅变法", Turn:1}
  4. 在句首补全事件主语 "商鞅变法"
  5. 置信度：1.0 × 0.85 = 0.85（恰好等于阈值，使用规则结果）

消解结果: "商鞅变法后来呢？"
置信度: 0.85
```

### 示例 3：LLM 处理复杂情况

```
对话历史：
  Turn 1 - 用户: "介绍一下孙武和孙膑"
  Turn 1 - 系统: "孙武是春秋时期的军事家，孙膑是战国时期的..."

Turn 2 - 用户输入: "他们两人有什么相同点和不同点？"

规则消解流程：
  1. 检测到 "他们" → 类型 person
  2. 实体栈 persons：[孙武(Turn:1), 孙膑(Turn:1)]
  3. 需要同时指代两个人 → 触发 "多人关系" 检测
  4. 置信度不足，NeedsLLM = true

LLM 消解：
  传入: 对话历史 + 当前问题
  LLM 输出:
  {
    "resolved_query": "孙武和孙膑有什么相同点和不同点？",
    "entities": {"他们两人": "孙武和孙膑"},
    "reasoning": "上文同时介绍了孙武和孙膑，'他们两人'指代这两位军事家"
  }

消解结果: "孙武和孙膑有什么相同点和不同点？"
置信度: 0.95
```

## 六、当前限制与后续优化方向

### 当前限制

| 限制 | 原因 | 影响 |
|------|------|------|
| 依赖实体词典 | jieba 实体识别基于预定义词典 | 词典外的实体无法识别 |
| 无深层句法分析 | 没有使用依存句法树 | 复杂嵌套结构可能出错 |
| LLM 调用有延迟 | 网络 + 推理时间 | 复杂情况响应变慢 |
| 跨轮距离有限 | 实体栈仅保留最近 10 条 | 超长对话可能丢失上下文 |

### 后续优化方向

**P1 - 实体权重优化**
```
权重计算：
  频率权重 = 提及次数 × 0.3
  距离权重 = 1 / (当前轮次 - 实体轮次 + 1) × 0.5
  位置权重 = 1 - (位置 / 句子长度) × 0.2

综合权重 = 频率权重 + 距离权重 + 位置权重
```

**P2 - 动词-实体类型约束**
```
通过动词推断代词的实体类型：
  "称霸" "伐" "攻" "联盟" → 主语更可能是人物或国家
  "爆发" "发生" "持续"    → 主语更可能是事件
  "开始" "结束" "延续"    → 主语更可能是时间段或事件
```

**P3 - 引入轻量 NER 模型**
```
使用预训练的中文 NER 模型（如 BERT-base-Chinese + NER fine-tune）
替代当前基于词典的实体识别，可以识别词典外的历史实体
模型大小：~400MB，推理时间：~50ms（GPU）/ ~200ms（CPU）
```

**P4 - 向量化实体匹配**
```
将实体嵌入为向量，通过语义相似度进行指代消解
可以处理同义表达："齐桓公" ≈ "姜小白" ≈ "春秋首霸"
与 Milvus 向量库集成，复用现有基础设施
```

## 八、调用方式

```go
// 创建查询优化器（内置指代消解）
optimizer := rag.NewQueryOptimizer()

// 多轮对话中使用
dialogHistory := []string{
    "用户: 介绍一下齐桓公",
    "系统: 齐桓公，春秋五霸之首...",
}

result, _ := optimizer.OptimizeQueryWithContext(
    "他是怎么称霸的？",  // 当前查询
    dialogHistory,      // 对话历史
    2,                  // 当前轮次
)

fmt.Println(result.OriginalQuery)   // "他是怎么称霸的？"
fmt.Println(result.ResolvedQuery)   // "齐桓公是怎么称霸的？"
fmt.Println(result.CorefConfidence) // 0.9

// 新对话时重置上下文
optimizer.ResetContext()
```
