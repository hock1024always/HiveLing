## 数据模型

```go
type Session struct {
```

```go
type Message struct {
```

## 消息标签系统

### 标签类型

| 标签 | 说明 | 示例 |
|------|------|------|
| `question` | 用户提问 | "齐桓公是谁？" |
| `answer` | AI 回答 | "齐桓公是春秋五霸之首..." |
| `follow_up` | 追问 | "那他有什么成就？" |
| `clarification` | 澄清说明 | "我指的是春秋时期..." |
| `fact` | 事实陈述 | "齐桓公在位时间是..." |
| `opinion` | 观点表达 | "我认为齐桓公是..." |
| `tool_call` | 工具调用 | [调用 search_knowledge] |
| `tool_result` | 工具结果 | [返回知识库结果] |
| `summary` | 摘要 | [对话摘要] |
| `greeting` | 问候 | "你好" |
| `thanks` | 感谢 | "谢谢" |
| `other` | 其他 | - |

### 标签识别规则

```go
func (m *SessionManager) classifyMessage(role, content, toolCalls string) string {
    // 1. 有工具调用 → tool_call
    if toolCalls != "" {
        return TagToolCall
    }

    // 2. 问候语检测
    if isGreeting(content) {
        return TagGreeting
    }

    // 3. 感谢语检测
    if isThanks(content) {
        return TagThanks
    }

    // 4. 问题检测（问号、疑问词）
    if isQuestion(content) {
        return TagQuestion
    }

    // 5. 追问检测
    if isFollowUp(content) {
        return TagFollowUp
    }

    // 6. 根据角色判断
    if role == "assistant" {
        return TagAnswer
    }

    return TagOther
}
```

## 智能压缩策略

### 压缩触发条件

- 消息数量 > 10 条时触发压缩
- 保留最近 5 条消息不参与压缩

### 压缩流程

## 上下文构建

### 消息选择策略

```go
func (m *SessionManager) BuildContext(sessionID string, maxMessages int) []Message {
    messages := getAllMessages(sessionID)
    
    if len(messages) <= maxMessages {
        return messages  // 无需压缩
    }

    // 智能选择
    selected := []Message{}
    
    // 1. 始终保留最近 5 条
    selected = append(selected, messages[len(messages)-5:]...)
    
    // 2. 选择高重要性消息
    for _, msg := range messages[:len(messages)-5] {
        if msg.Importance >= 8 || msg.Tag == "question" {
            selected = append(selected, msg)
        }
    }
    
    // 3. 填充剩余空间
    // ...
    
    return sortByTime(selected)
}
```

## 性能优化

### 1. 缓存策略

- 会话信息缓存: 30 分钟
- 消息添加后自动清除缓存

### 2. 批量操作

- 实体提取使用预编译正则
- 重要性评估避免重复计算

### 3. 延迟压缩

- 只在需要构建上下文时才触发压缩
- 避免频繁的摘要生成

## 使用示例

### 添加消息（自动标签）

```go
sessionMgr := memory.NewSessionManager()
```

## 总结

通过引入消息标签、重要性评估和智能压缩策略，上下文记忆系统能够：

✅ **智能分类** - 自动识别消息类型  
✅ **保留关键** - 高重要性消息优先保留  
✅ **高效压缩** - 结构化摘要保留核心信息  
✅ **实体追踪** - 自动提取和聚合关键实体  
✅ **话题管理** - 追踪对话主题演变  

这种设计特别适合长对话场景，能够在有限的上下文窗口内保留最有价值的信息。
